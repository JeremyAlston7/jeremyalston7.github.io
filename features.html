Gradient Text / Card Borders

A radial gradient follows your cursor along the card border by using JavaScript to track mouse position on each card. Every mouse movement recalculates the cursor's position relative to the card using getBoundingClientRect(), then updates a CSS radial-gradient background centered at that exact point. The illusion of a border is created by a 4px gap between the outer gradient div and a dark inner div.

Typing & Backspacing Animation

JavaScript cycles through an array of preset strings, adding one character at a time using string slicing and a recursive setTimeout loop. Once the full string is typed, a pause is triggered before the deletion phase begins, removing one character at a time at a faster speed. When the string is fully deleted, the index moves to the next string and the cycle repeats.

Slot Machine Animation

A span element's text content is swapped on a set interval using setInterval. Before swapping, a CSS slideOut keyframe animation plays, moving the text upward. After a short delay, the text is updated and a slideIn animation plays from below, creating the illusion of a slot machine rolling to the next value.

Wave Text Animation

JavaScript splits the text content into individual characters and wraps each one in its own span element. The same CSS keyframe animation is applied to every span, but each one has a slightly larger animation-delay than the previous one. This staggered delay creates a smooth ripple wave effect across the full line of text.

Animated Orb Background

Three empty div elements are given a large width and height, a border-radius of 50% to make them circular, and a heavy CSS blur filter to create a soft glowing effect. A looping CSS keyframe animation using translate moves each orb slowly in a different direction. The alternate keyword reverses the animation each cycle, making the orbs drift back and forth smoothly forever.

Proficiency Bars Animation

Each bar starts with a width of 0 set in CSS. The IntersectionObserver API watches the bar section and when it enters the viewport, JavaScript reads a data-width attribute from each bar element and sets that as the inline width. A CSS transition on the width property smoothly animates the bar growing from 0 to its target percentage. When the section leaves the viewport the width resets to 0, allowing the animation to replay on the next scroll.
